# Architecture Decision Records (ADRs)

## Intent

Track key decisions, tradeoffs, and their rationale.

## When to write an ADR

- Choosing a major technology or framework
- Defining service boundaries and APIs
- Selecting data models or storage approaches
- Security and compliance decisions

## Template

```
# ADR-XXXX: Title

## Status

Proposed | Accepted | Rejected | Superseded

## Context

What problem are we solving? What constraints exist?

## Decision

What did we decide?

## Alternatives

What other options were considered and why were they rejected?

## Consequences

What are the tradeoffs and follow-up actions?
```
